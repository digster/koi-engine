## Key decisions

- **Prepare before the passes.** `uploadToGpuBuffer` acquires + submits its OWN command buffer, and command buffers
  execute in submission order, so instance uploads done before the frame cmd is submitted are guaranteed ready. To
  stay clean (and never open a copy pass while a render pass is active), ALL culling/sorting/packing/uploading moved
  into `buildFrameBatches`, run up front in `renderSceneAndPost` — the record functions became pure submit. This is
  *traverse → list → **batch** → submit*.
- **Two batch keys.** Colour needs `(material, mesh)` (shared textures AND geometry). Shadow is depth-only →
  material-blind → keys on **mesh alone** (bigger runs; demo shadow 11→5 vs colour 11→8). Shadow is still never
  camera-culled (off-screen casters shadow in-view geometry).
- **Transform → instance buffer.** A uniform is constant per-draw, useless when one draw covers many objects, so
  `model`/`normalMatrix` became per-instance attributes and the uniform shrank to the shared `viewProj`. Per-instance
  `normalMatrix` is required because the demo satellites are non-uniformly scaled.
- **Transparent stays unbatched.** Alpha blending is order-dependent, so translucent items keep back-to-front order
  and draw one-instance-each at their own slot (packed after the opaque block in the same instance buffer).
