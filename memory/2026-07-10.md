## Key decisions

- **UI is its own pass, in LDR, drawn last.** The single most important choice. Debug lines draw *into* the HDR
  scene target and get tone-mapped + FXAA'd; that would blur/desaturate **text**. So the HUD draws in a separate
  pass on the **final image after post** (`LOAD_OP_LOAD`, alpha blend, no depth) at the swapchain format ‚Üí crisp
  glyphs. Contrast this with debug draw whenever adding future overlays.
- **One pipeline for text + panels.** A reserved solid-white atlas cell means `rect()` and `text()` share the same
  textured pipeline and draw together ‚Äî glyphs sample coverage (white-on-transparent, alpha = coverage), panels
  sample the white cell; `frag = atlas √ó tint` handles both.
- **Atlas bleeding is real.** Nearest filtering + edge-to-edge cells ‚Üí a sample at a cell boundary can `floor()`
  into the neighbour (descenders bleeding down). Fixed with a **half-texel UV inset** in `cellUV`. Classic atlas
  gotcha; noted in code + doc.
- **shared_ptr members need explicit teardown.** `hudAtlas_` (a `Texture`) must be `.reset()` in the destructor
  body *before* `SDL_DestroyGPUDevice`, exactly like `skyboxMesh_` ‚Äî member destructors run after the body.
