# 2026-06-28 — Step 0: Project foundation + window & clear screen

## What we did

Bootstrapped the **Koi Engine** repo from empty: a learning-focused 3D engine in
C++20 on SDL3's GPU API. Completed **Step 0** — a window that clears the screen
each frame, plus the full project scaffolding the rest of the engine grows from.

## Key decisions

- **Renderer:** SDL3 **GPU API** (Metal on macOS, Vulkan/D3D12 elsewhere), chosen
  over OpenGL (deprecated on macOS). Confirmed working on the **Metal** backend.
- **Math:** will be **hand-rolled** (no GLM), starting Step 3.
- **Step 0 scope:** window + clear only — the shader toolchain/triangle is Step 1.
- **Docs philosophy:** `docs/` are written as a **beginner's graphics guide**
  (concepts from first principles), not just code commentary. Captured in the
  project `CLAUDE.md`.

## What was built

- **Build:** `CMakeLists.txt` (C++20; `koi_core` static lib + `koi-engine` app +
  `koi-tests`; strict warnings via an INTERFACE target) and `CMakePresets.json`
  (debug/release, Unix Makefiles). doctest pulled as a single downloaded header
  (its CMake project is incompatible with CMake 4.x).
- **Engine source** (`src/`, all `namespace koi`, heavily commented):
  - `core/Log.hpp` — leveled logging macros over SDL (`SDL_LOG_CATEGORY_APPLICATION`).
  - `core/Window.*` — RAII `SDL_Window` wrapper.
  - `renderer/GpuRenderer.*` — owns `SDL_GPUDevice` + swapchain; `renderFrame()`
    does acquire cmd buffer → acquire swapchain → render pass (clear) → submit.
  - `core/Engine.*` — owns subsystems, runs the game loop, dispatches events.
  - `main.cpp` — tiny entry point.
- **Docs:** `docs/00-getting-started.md` (build/run/test/layout) and
  `docs/01-window-and-render-loop.md` (GPU/swapchain/command buffer/render pass
  explained for beginners, mapped to the code).
- **Meta:** README, ARCHITECTURE, CLAUDE, LICENSE (MIT/digster), .gitignore,
  PROMPT.md, this memory file, `tests/test_core.cpp`.

## Gotchas hit & fixed

- **`unique_ptr` + forward-declared types:** `Engine`'s ctor/dtor must be defined
  in the `.cpp` (not `= default` in the header) so the complete `Window`/`GpuRenderer`
  types are visible to `unique_ptr`'s destructor.
- **Logging after `SDL_Quit`:** `SDL_Quit` resets log priorities; a custom log
  category defaults to ERROR and silently dropped our shutdown INFO line. Fixed by
  using `SDL_LOG_CATEGORY_APPLICATION` and logging before `SDL_Quit`.
- **doctest vs CMake 4.x:** its bundled `cmake_minimum_required` is too old —
  download the header directly instead of building its project.

## Step 1 — First triangle (shaders + graphics pipeline)

Drew the RGB-gradient triangle with our own GLSL shaders.

**Decisions:**
- **Toolchain pivot:** the roadmap assumed HLSL + SDL `shadercross`, but the
  installed `shadercross` CLI is **broken** (`dyld: Library not loaded:
  @rpath/libSDL3_shadercross.0.dylib ... no LC_RPATH's found`) and HLSL→SPIR-V
  needs DXC (not installed). Pivoted to **GLSL → `glslc` → SPIR-V → `spirv-cross`
  → MSL** — both tools installed and verified; also more beginner-friendly.
- Triangle data is **baked into the vertex shader** (indexed by `gl_VertexIndex`),
  no vertex buffer yet (that's Step 2).

**Built:**
- `shaders/triangle.{vert,frag}` (GLSL 450). CMake `koi_add_shader()` runs glslc +
  spirv-cross into `build/shaders/`; `koi_shaders` target; `koi-engine` depends on it.
- `src/renderer/Shader.{hpp,cpp}`: `selectShaderVariant` (pure: format→ext+entry;
  MSL entry is `main0`) + `loadShader` (loads from `SDL_GetBasePath()/shaders/`).
- `GpuRenderer`: builds the graphics pipeline in the ctor (color target format =
  `SDL_GetGPUSwapchainTextureFormat`, `TRIANGLELIST`), binds + `SDL_DrawGPUPrimitives(3,1,0,0)`
  in `renderFrame`, releases pipeline in dtor; `isValid()` now also checks the pipeline.
- `tests/test_shader.cpp`: doctest cases for `selectShaderVariant`.
- `docs/02-first-triangle.html` beginner guide; nav + roadmap updated across docs.
- Meta: README/ARCHITECTURE/CLAUDE updated; `.env.example` added (KOI_MAX_FRAMES).

**Gotcha:** spirv-cross renames the MSL entry point `main` → `main0`; the loader
sets entry point per-format accordingly.

**Verified:** clean warning-free build (shaders compile via glslc/spirv-cross);
`build/shaders` has 4 files; `ctest` → 7 cases / 19 assertions pass; `KOI_MAX_FRAMES=60`
run loads MSL shaders (`entry 'main0'`), "Graphics pipeline ready", clean exit 0,
no GPU validation errors. (Live-window screenshot skipped — user declined screen
access; deterministic checks cover correctness.)

## Update — headless render-to-image capture (KOI_CAPTURE)

Added a visual-debugging tool (user request) so we can verify rendering without
screen-recording access:

- `GpuRenderer::captureFrame(path, clearColor)`: renders one frame into an
  off-screen `R8G8B8A8`/`B8G8R8A8` texture (same format as the swapchain so the
  existing pipeline works), downloads it via a copy pass + download transfer buffer,
  waits on a fence, wraps the pixels in an `SDL_Surface`, and `SDL_SaveBMP`s them.
- Triggered by `KOI_CAPTURE=<path.bmp>` in `Engine::run` (one-shot, then exit).
- Pure helper `gpuColorFormatToPixelFormat` (GPU format → SDL byte-order pixel
  format) with unit tests in `test_core.cpp`.
- BMP chosen because SDL core can save it with no extra deps; convert with
  `sips -s format png` if PNG is wanted.
- Documented in `CLAUDE.md` (Visual debugging), `.env.example`, `ARCHITECTURE.md`,
  and `docs/00-getting-started.html`.
- **Verified:** captured the first triangle (red/green/blue gradient on dark teal) —
  output is correct. Tests: 8 cases / 22 assertions pass.

## Step 2 — Vertex & index buffers (draw a quad)

Moved geometry out of the shader into real GPU buffers; upgraded the triangle to an
RGB-gradient **quad** (user chose quad over triangle so the index buffer is meaningful
— 4 vertices reused by 6 indices).

**Built:**
- `shaders/triangle.vert`: now takes `in vec2 inPosition` (loc 0) + `in vec3 inColor`
  (loc 1) instead of hardcoded arrays / `gl_VertexIndex`. `triangle.frag` unchanged.
- `src/renderer/Vertex.hpp` (new): `koi::Vertex { float position[2]; float color[3]; }`,
  `static_assert`s pinning sizeof==20 / offsets 0,8 (the pipeline's attribute contract).
- `GpuRenderer`:
  - `uploadToGpuBuffer(usage, data, size)` — the two-step upload (create GPU buffer →
    create+map UPLOAD transfer buffer → memcpy → copy pass → submit). Mirror of
    `captureFrame`'s download. No fence: one-time init, command buffers run in order.
  - `createGeometry()` — 4 `Vertex` corners + 6 `Uint16` indices (`0,1,2, 2,3,0`),
    uploaded into `vertexBuffer_` / `indexBuffer_`; `indexCount_ = 6`.
  - `createTrianglePipeline()` — added `vertex_input_state`: one buffer desc
    (slot 0, pitch=sizeof(Vertex), per-vertex) + two attributes (FLOAT2@0, FLOAT3@8).
  - `recordQuad(pass)` — shared bind+draw helper (pipeline + vertex/index buffers +
    `SDL_DrawGPUIndexedPrimitives`); called by both `renderFrame` and `captureFrame`
    so the live and capture paths can't drift. Buffers released in dtor; `isValid()`
    now also checks them.
- `tests/test_vertex.cpp` (new, registered in CMake): sizeof/offset + quad-index guards.
- `docs/03-vertex-and-index-buffers.html` (new beginner guide: CPU vs GPU memory,
  staging buffer + copy pass, vertex input layout & the 4-layer `location` chain,
  why index buffers exist). Nav/roadmap/cards updated across docs + README; ARCHITECTURE
  updated (Vertex row, geometry subsection, frame flow). `main.cpp` title → Step 2.

**Kept names** `trianglePipeline_` / `createTrianglePipeline` and the `triangle.*`
shader filenames (limit churn; shaders still named triangle.*). No new env vars.

**Gotcha:** `-Wconversion` — `pitch = sizeof(Vertex)` (size_t→Uint32) needs an explicit
cast; the `offsetof` values in the attribute brace-init are constant-expressions that
fit, so no cast needed there.

**Verified:** clean warning-free build; shaders recompiled (the `.vert.msl` now shows
`inPosition [[attribute(0)]]` / `inColor [[attribute(1)]]`); `ctest` → 11 cases /
33 assertions pass; `KOI_MAX_FRAMES=5` smoke clean (logs "Geometry uploaded"), exit 0,
no GPU validation errors; `KOI_CAPTURE` → correct centered RGB/yellow gradient quad
(visible diagonal seam between the two triangles; stretched to a rectangle because
aspect-ratio correction waits for Step 3's projection matrix).

## Step 3 — Spinning 3D cube (hand-rolled math, MVP uniform, depth buffer)

Turned the flat 2D quad into a rotating 3D RGB cube — the engine's first real math and
the core 3D transform pipeline.

**Decisions (user):** cube **auto-spins** (per-frame MVP from `SDL_GetTicks()`; capture
uses a fixed angle). **Quaternions deferred** (no use case — "introduce when needed"
beats CLAUDE.md's parenthetical list); Step 3 math is `Vec` + `Mat4` only. **View matrix
deferred to Step 4** (camera): V = identity, camera distance folded into the model
translate, so MVP = Projection · Model.

**Built:**
- `src/math/Vec.hpp` (Vec2/3/4 + dot/cross/length/normalize) and `src/math/Mat4.hpp`
  (column-major `float[16]`; identity, mul, mat·vec, translation, rotationX/Y/Z,
  perspective). Header-only (small/pure/testable — a deliberate exception to the .hpp/.cpp
  split). `perspective` is right-handed, depth **z∈[0,1]** (SDL3/Metal convention), +Y up.
- `shaders/triangle.vert`: 3D `inPosition`, `layout(set=1,binding=0) uniform UBO { mat4 mvp; }`,
  `gl_Position = mvp * vec4(inPosition,1)`. Frag unchanged.
- `Shader.{hpp,cpp}`: `loadShader` gained `numUniformBuffers` param (vertex = 1).
- `Vertex.hpp`: position widened to `float[3]` → sizeof 24, color offset 12 (asserts + tests updated).
- `GpuRenderer`: cube geometry (8 verts / 36 indices, RGB-cube coloring); position attr
  FLOAT2→FLOAT3; `chooseDepthFormat` (D32_FLOAT→D24→D16 via SupportsFormat); `depthTexture_`
  + `ensureDepthTexture(w,h)` (resizes); depth_stencil_state (LESS, test+write) and
  depth target in pipeline; `buildMvp(aspect, angle)` = perspective(60°)·translate(0,0,-3)·
  rotY·rotX; per-frame `SDL_PushGPUVertexUniformData(cmd,0,&mvp,64)`; `recordQuad`→`recordCube`;
  depth texture released in dtor. `renderFrame` now captures swapchain w/h for depth size + aspect.
- `tests/test_math.cpp` (new; vec + mat4 assertions incl. perspective near/far → 0/1,
  rotZ 90° maps +X→+Y) registered in CMake; `tests/test_vertex.cpp` updated (sizeof 24, cube indices).
- Docs: new `docs/04-3d-cube-mvp-and-depth.html` (spaces/MVP, homogeneous coords + w-divide,
  column-major, uniform buffers + set=1, depth buffer). Nav/roadmap/cards across docs + README;
  ARCHITECTURE (math subsystem row, MVP/depth subsection, frame flow). `main.cpp` title → Step 3.

**Gotchas hit & fixed:**
- **GPU hang** on first run: a zeroed `SDL_GPUDepthStencilTargetInfo` leaves the stencil
  load op = `LOAD` (enum value 0), and SDL forbids `cycle=true` with a LOAD op → debug-layer
  assertion, GPU never idle, shutdown's wait-for-idle hung. Fix: set `stencil_load_op` /
  `stencil_store_op` = `DONT_CARE` explicitly (our D32_FLOAT has no stencil). Lesson: in this
  API "all zeros" is a real config, not "unset".
- **spirv-cross MSL interop checkpoint passed:** the manual GLSL→SPIR-V→MSL toolchain put the
  vertex uniform at `[[buffer(0)]]` and vertex inputs via `[[stage_in]]` by default — exactly
  SDL3's Metal convention, so no extra spirv-cross flags were needed.

**Verified:** clean warning-free build; MSL shows `[[buffer(0)]]` uniform + `float3 inPosition`;
`ctest` → 19 cases / 104 assertions pass; `KOI_MAX_FRAMES` smoke clean, exit 0, no validation
errors; `KOI_CAPTURE` → correct solid 3D RGB cube, 3 faces, proper depth occlusion, square
proportions (Step 2 stretch fixed), right-side-up & in front of camera (confirms +Y up, RH view,
z∈[0,1], aspect — all in one image).
