# 2026-06-28 — Step 0: Project foundation + window & clear screen

## What we did

Bootstrapped the **Koi Engine** repo from empty: a learning-focused 3D engine in
C++20 on SDL3's GPU API. Completed **Step 0** — a window that clears the screen
each frame, plus the full project scaffolding the rest of the engine grows from.

## Key decisions

- **Renderer:** SDL3 **GPU API** (Metal on macOS, Vulkan/D3D12 elsewhere), chosen
  over OpenGL (deprecated on macOS). Confirmed working on the **Metal** backend.
- **Math:** will be **hand-rolled** (no GLM), starting Step 3.
- **Step 0 scope:** window + clear only — the shader toolchain/triangle is Step 1.
- **Docs philosophy:** `docs/` are written as a **beginner's graphics guide**
  (concepts from first principles), not just code commentary. Captured in the
  project `CLAUDE.md`.

## What was built

- **Build:** `CMakeLists.txt` (C++20; `koi_core` static lib + `koi-engine` app +
  `koi-tests`; strict warnings via an INTERFACE target) and `CMakePresets.json`
  (debug/release, Unix Makefiles). doctest pulled as a single downloaded header
  (its CMake project is incompatible with CMake 4.x).
- **Engine source** (`src/`, all `namespace koi`, heavily commented):
  - `core/Log.hpp` — leveled logging macros over SDL (`SDL_LOG_CATEGORY_APPLICATION`).
  - `core/Window.*` — RAII `SDL_Window` wrapper.
  - `renderer/GpuRenderer.*` — owns `SDL_GPUDevice` + swapchain; `renderFrame()`
    does acquire cmd buffer → acquire swapchain → render pass (clear) → submit.
  - `core/Engine.*` — owns subsystems, runs the game loop, dispatches events.
  - `main.cpp` — tiny entry point.
- **Docs:** `docs/00-getting-started.md` (build/run/test/layout) and
  `docs/01-window-and-render-loop.md` (GPU/swapchain/command buffer/render pass
  explained for beginners, mapped to the code).
- **Meta:** README, ARCHITECTURE, CLAUDE, LICENSE (MIT/digster), .gitignore,
  PROMPT.md, this memory file, `tests/test_core.cpp`.

## Gotchas hit & fixed

- **`unique_ptr` + forward-declared types:** `Engine`'s ctor/dtor must be defined
  in the `.cpp` (not `= default` in the header) so the complete `Window`/`GpuRenderer`
  types are visible to `unique_ptr`'s destructor.
- **Logging after `SDL_Quit`:** `SDL_Quit` resets log priorities; a custom log
  category defaults to ERROR and silently dropped our shutdown INFO line. Fixed by
  using `SDL_LOG_CATEGORY_APPLICATION` and logging before `SDL_Quit`.
- **doctest vs CMake 4.x:** its bundled `cmake_minimum_required` is too old —
  download the header directly instead of building its project.

## Testing affordance added

`KOI_MAX_FRAMES` env var renders N frames then exits via the normal shutdown path
— makes headless smoke tests deterministic.

## Step 1 — First triangle (shaders + graphics pipeline)

Drew the RGB-gradient triangle with our own GLSL shaders.

**Decisions:**
- **Toolchain pivot:** the roadmap assumed HLSL + SDL `shadercross`, but the
  installed `shadercross` CLI is **broken** (`dyld: Library not loaded:
  @rpath/libSDL3_shadercross.0.dylib ... no LC_RPATH's found`) and HLSL→SPIR-V
  needs DXC (not installed). Pivoted to **GLSL → `glslc` → SPIR-V → `spirv-cross`
  → MSL** — both tools installed and verified; also more beginner-friendly.
- Triangle data is **baked into the vertex shader** (indexed by `gl_VertexIndex`),
  no vertex buffer yet (that's Step 2).

**Built:**
- `shaders/triangle.{vert,frag}` (GLSL 450). CMake `koi_add_shader()` runs glslc +
  spirv-cross into `build/shaders/`; `koi_shaders` target; `koi-engine` depends on it.
- `src/renderer/Shader.{hpp,cpp}`: `selectShaderVariant` (pure: format→ext+entry;
  MSL entry is `main0`) + `loadShader` (loads from `SDL_GetBasePath()/shaders/`).
- `GpuRenderer`: builds the graphics pipeline in the ctor (color target format =
  `SDL_GetGPUSwapchainTextureFormat`, `TRIANGLELIST`), binds + `SDL_DrawGPUPrimitives(3,1,0,0)`
  in `renderFrame`, releases pipeline in dtor; `isValid()` now also checks the pipeline.
- `tests/test_shader.cpp`: doctest cases for `selectShaderVariant`.
- `docs/02-first-triangle.html` beginner guide; nav + roadmap updated across docs.
- Meta: README/ARCHITECTURE/CLAUDE updated; `.env.example` added (KOI_MAX_FRAMES).

**Gotcha:** spirv-cross renames the MSL entry point `main` → `main0`; the loader
sets entry point per-format accordingly.

**Verified:** clean warning-free build (shaders compile via glslc/spirv-cross);
`build/shaders` has 4 files; `ctest` → 7 cases / 19 assertions pass; `KOI_MAX_FRAMES=60`
run loads MSL shaders (`entry 'main0'`), "Graphics pipeline ready", clean exit 0,
no GPU validation errors. (Live-window screenshot skipped — user declined screen
access; deterministic checks cover correctness.)
