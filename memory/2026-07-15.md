# 2026-07-15 — Step 25 — glTF node hierarchy & multi-material meshes

## What we did

Taught the glTF loader to import a whole **scene** instead of just the first mesh.
This was the roadmap's `◀── next` item ("glTF node hierarchy / Sponza"). Per the
project owner's decision we kept the step **small**: the loader extension only,
demonstrated on a small **generated** model — **Sponza is deferred** to the next
step (and with it the sampler/wrap-mode, data-URI, and draw-call-stress work Sponza
forces).

The key realization: the engine side already draws a deep, many-material tree
(`Node` propagates `world = parent·local`, `buildRenderQueue` flattens it, Step 24
instancing keys on `shared_ptr` identity). So **no `Node`/`RenderQueue`/`GpuRenderer`
changes** — the work is loader-side plus two small pure-math additions.

## New behaviour

- **`loadModelHierarchy(renderer, path)`** (new public entry in `ModelLoader`): walks
  the glTF default scene's node tree and returns a `std::unique_ptr<Node>` subtree.
  Each glTF node → a group `Node` carrying its local transform; each mesh
  **primitive** → one child draw-node (mesh + its own material). Recurses into child
  nodes. `loadModel`/`LoadedModel` is kept for the single-mesh path (sphere/torus/OBJ).
- **Node transforms**: read as explicit **TRS** (the rotation's `xyzw` already matches
  our `Quat` storage order, so it drops straight in) or, when the node is a raw 4×4
  **matrix**, decomposed back to TRS.
- **Texture dedup**: a per-import `std::unordered_map<const cgltf_image*,
  shared_ptr<Texture>>` threaded through `loadGltfImage`/`loadGltfMaterial`, so an
  image shared by many materials is uploaded **once**. (The log prints "N unique
  texture(s)".)

## New pure math (headless-testable, the riskiest bit)

- **`Quat::fromRotationMatrix(const Mat4&)`** in `src/math/Quat.hpp` — inverse of
  `toMat4()`. Trace-based (Shoemake): solves for whichever of x/y/z/w is provably
  largest first, so it stays accurate near 180° (where the naïve `w=½√(1+trace)` dies).
  Signs pinned to `toMat4()`.
- **`Transform::fromMatrix(const Mat4&)`** in `src/scene/Transform.hpp` — translation
  from the last column; scale = length of each basis column; rotation = the leftover
  orthonormal basis → `fromRotationMatrix`. Handles **mirrored** (negative-determinant)
  nodes by folding the flip into `scale.x`. Safe because glTF forbids shear in node
  matrices.
- Round-trip tests added to `tests/test_quat.cpp` + `tests/test_transform.cpp`
  (incl. non-uniform + mirrored scale). All **127** test cases pass.

## Refactor

Extracted the per-primitive vertex/index assembly out of the old single-mesh
`loadGltf` into **`loadPrimitiveMesh(renderer, prim)`**, reused by both the
single-mesh and hierarchy paths (still fills missing normals/tangents via the
existing helpers). `loadGltfImage`/`loadGltfMaterial` gained a `TextureCache&` param.

## Next

**Sponza** is the marked next milestone — it will force glTF **samplers/wrap modes**
(tiled textures need REPEAT; today one shared renderer-owned sampler), possibly
**data-URI** images, and real draw-call/culling stress. Also still deferred: material
**alpha-mode** import from glTF. After Sponza: alpha-tested cutout, cascaded shadows,
skeletal animation (which imports a joint hierarchy the same way).
