## Key decisions

- **Split-sum approximation** (Karis/Epic 2013). Three maps baked from the skybox cubemap:
  - **diffuse irradiance** cube (32², cosine-convolved hemisphere) — sampled by the normal.
  - **specular prefilter** cube (128², 5 roughness **mips**; mip = roughness via `textureLod`) — GGX
    importance-sampled; samples the env's own mips by pdf to kill fireflies.
  - **BRDF LUT** (512², RG16F) — the split-sum's **environment-independent** factor, so baked ONCE in
    `createIblResources` (never re-baked); the two cubes bake in `bakeIbl` from whatever sky is loaded.
- **Bake by rendering into cube faces** — `bakeIbl` draws the cube six times per target (once per face,
  and per mip for the prefilter) with a 90° camera down each axis (`ibl_cube.vert`, `cubeCaptureViews`).
  `SDL_GPUColorTargetInfo.layer_or_depth_plane` + `.mip_level` select the subresource. **No `Mat4`
  inverse needed** (the gap Step 14 hit) — the cube corner's object-space position IS the world sample
  direction, so `lookAt`+`perspective` suffice.
- **Bake-at-load, not per-frame.** `bakeIbl` is called at the end of `loadCubemap` (a sky reload
  re-bakes automatically). `iblReady_` gates use until a bake ran; `iblEnabled_` (key **9**) is the
  runtime A/B toggle, passed to the shader in the light UBO's spare **`ambient.w`** lane (no UBO layout
  change). Shader uses a **uniform if-branch** on `ambient.w` (IBL vs flat fill) — safe for the
  cubemap's implicit-derivative sampling, and avoids NaN-from-unbaked-maps.
- **IBL geometry term uses `k = roughness²/2`**, NOT the direct-lighting `(r+1)²/8` — a classic silent
  IBL bug, so it's a separate function (`geometrySmithIBL`) in both the shader and the CPU mirror.
- **Demo showcase:** the smooth metal sphere (roughness lowered 0.20→0.08). Already metallic, so IBL's
  effect is immediate: dark ball → mirrors the sky.
