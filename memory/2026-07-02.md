# 2026-07-02 — ROADMAP.md production-readiness review (analysis only)

## What we did

The user stated a new long-term goal: eventually take Koi to a **professional level and use it for

untouched (only PROMPT.md + this memory file were written).

## Findings reported (candidate ROADMAP.md improvements)

1. **The defining milestone is missing: engine/app separation.** Today `Engine::buildScene`
   hardcodes the demo scene and `main.cpp` is the app. "Production apps" requires Koi to become a
   consumable library with a public API boundary, a `samples/` or sandbox app, and eventually
   versioning. Nothing on the roadmap names this.
2. **Flag "architecture pivots" separately from additive features** — things cheap to plan early
   and expensive to retrofit: multithreading/job system (absent entirely), render-queue extraction
   (decouple scene traversal from draw submission; prerequisite for sorting/culling/instancing
   already on the roadmap), shader variant/permutation management (the single uber
   `triangle.frag` grows per feature; normal-maps-optional + skinning + deferred will force it),
   and a memory strategy (everything is `shared_ptr`/`unique_ptr`; no allocator/streaming plan).
3. **Promote cross-platform validation + CI from "Platform & build" (far out) to near-term.**
   Only macOS/Metal is exercised; backend-specific bugs accumulate (the Step 9 spirv-cross
   `--msl-decoration-binding` swap is the existing proof). A Linux/Vulkan CI build (lavapipe) +
   `KOI_CAPTURE`-based **golden-image regression tests** is the highest-leverage production item
   and builds on infrastructure that already exists.
4. **Asset pipeline (offline cooking) missing** — runtime asset manager is listed, but not an
   offline import→process→pack step (engine-native formats, KTX2 fits here, `gen_models.py` is a
   seed).
5. **Robustness track missing:** defined failure policy (missing assets, device lost), fuzzing the
   OBJ/glTF loaders (untrusted input), sRGB/color-management correctness (albedo currently treated
   as linear — documented shortcut).
6. **Roadmap-document structure:** add a "definition of 1.0" / quality-gates section; tag items by
   phase (learning → hardening → production); fixed-timestep loop should precede physics; small
   duplication — "asset hot-reload" appears in both Scene & data architecture and Platform & build.

## Next

phase tags), add the engine/app-separation milestone, and move CI + golden-image testing up.
No code change implied yet.
