  (center/extents/contains/expand/merge/`transformed`, with an inverted `empty()` identity), `Plane`

  `intersect(ray, box, tHit)`.
- **Normal matrix** wired through [`shaders/triangle.vert`](../shaders/triangle.vert) + the per-draw
  `VertexUniform` in [`GpuRenderer.cpp`](../src/renderer/GpuRenderer.cpp): the vertex UBO grew to
  `{ mvp, model, normalMatrix }`, normals now use `mat3(normalMatrix)` = `transpose(inverse(model))`.
- Tutorial [`documentation/docs/20-geometry-utilities.html`](../documentation/docs/20-geometry-utilities.html)
  + index nav/card; tests [`tests/test_geometry.cpp`](../tests/test_geometry.cpp).

## Key decisions

- **Frustum extraction is convention-sensitive.** Gribb–Hartmann textbook formula assumes OpenGL z∈[-1,1]
  (near = row3+row2). Koi uses **z∈[0,1]** (Vulkan/D3D/Metal), so the **near plane is `row2` alone**. Getting
  this wrong makes objects vanish at the near plane — called out loudly in code + doc + a dedicated test.
- **Normal vs tangent asymmetry.** Only the *normal* (a covector) moves to the normal matrix; the **tangent
  stays on `mat3(model)`** (it's a true surface direction), and the fragment shader re-orthonormalizes the
  pair. For an affine matrix, `mat3(transpose(inverse(model)))` is exactly the classic 3×3 normal matrix (the
  translation coupling lands in the 4th column), so we reuse the full 4×4 inverse.
- **Kept it pure math.** No render-queue refactor, no actual draw-culling, no picking UI — those *consume*
  this layer and are later steps. Mirrors Step 18 (build+test the primitive; visible payoff comes later),
  except we added the normal-matrix fix as a live payoff this time.

- Demo scene is **uniform-scale** only, so the `KOI_CAPTURE` frame is **visually unchanged** (normalized normal

# 2026-07-04 — Step 20: Render queue extraction + frustum culling

## What we did

Completed **Step 20**. Started from "pick up something from the roadmap to work on next." The roadmap flagged
**CI + golden images** as the highest-leverage next move; presented it, but the user twice asked for **non-CI
options** and chose **render queue + frustum culling** — the architecture pivot that consumes the Step 19

tracks). Turned the inline, recursive draw path into **traverse → flat list → submit**, then added

## Key decisions

- **New `src/renderer/RenderQueue.hpp` + `.cpp`.** `RenderItem { const Mesh*, const Material*, Mat4 world,
  Aabb worldBounds }`; `computeLocalBounds(span<Vertex>)` + `cullToFrustum(...)` are **inline & pure** (unit-
  tested, no GPU); `buildRenderQueue(root, out)` needs the full Node/Mesh so it lives in the `.cpp`. Header
  forward-declares Mesh/Material/Node; includes Geometry/Vertex/Mat4.
- **`Mesh` caches a model-space `Aabb`.** Computed in `createMeshImpl` from the vertex span **before upload**
  (the only moment the CPU vertices exist), passed to the ctor (new param before the defaulted `elemSize`).
  World box = `mesh->localBounds().transformed(world)`, built into each item — cached so culling is a pure
  plane test with no per-frame rebuild.
- **Both passes consume one queue.** `renderSceneAndPost` builds `renderQueue_` once. **Shadow pass loops the
  WHOLE queue** — deliberately NOT camera-culled: an off-screen caster can still cast an in-view shadow
  (culling it makes shadows blink). **Camera pass** builds `Frustum::fromViewProjection(projView)` from the
  exact same `projView` the shader uses, filters into `visibleItems_`, submits survivors. The subtle
  shadow-pass-culling trap is called out in code, the doc, and ARCHITECTURE.
- **`recordNode`/`recordShadowNode` recursion → per-item `submitItem` + a shadow loop.** Binding/uniform logic
  moved verbatim; `submitItem` reads `item.world` instead of `node.worldMatrix()`.
- **Const-correctness gotcha:** `recordScene` is `const` but fills the `visibleItems_` scratch → made that
  member `mutable` (classic const-method cache). `renderQueue_` stays non-mutable (built in the non-const
  `renderSceneAndPost`).
- **Runtime toggle key `0`** (`setFrustumCullingEnabled`) + a throttled `drew N / total (culled M)` log so the
  win is observable; culling **on** by default.

# 2026-07-04 — Step 21: Transparency + alpha blending

## What we did

Picked the roadmap's stated next milestone ("transparency + blending ◀── next"). Added the engine's first
**translucent surfaces**:

- **`Material`** ([src/scene/Material.hpp](../src/scene/Material.hpp)): new `enum class AlphaMode { Opaque,
  Blend }` + `alphaMode`/`opacity` fields, appended **last** so the positional aggregate inits elsewhere keep
  working.
- **Shader** ([shaders/triangle.frag](../shaders/triangle.frag)): sample albedo once as a `vec4`, output
  `outColor = vec4(color, material.w * albedoSample.a)` (was hardcoded `1.0`). `material.w` is the opacity,
  pushed via the previously-spare lane of the material uniform in `submitItem`.
- **Second pipeline** ([src/renderer/GpuRenderer.cpp](../src/renderer/GpuRenderer.cpp)): `createTrianglePipeline`
  now builds `transparentPipeline_` from the *same* shaders/layout with **blend on** (SRC_ALPHA / 1−SRC_ALPHA,
  ADD) and **depth-write off** (test stays LESS). Released in the dtor; added to `isValid()`.
- **Pass split**: extracted `bindFrameLighting` (shadow slot 4 + IBL slots 5–7 + light uniform) so it can re-run
  after a pipeline switch. `recordScene` now: cull → `partitionByBlend` → **opaque** (trianglePipeline) → **sky**
  → **transparent** (transparentPipeline, sorted). The **skybox moved before** the transparent pass — it writes
  no depth, so glass drawn first would be painted over by the sky.
- **`partitionByBlend`** ([src/renderer/RenderQueue.cpp](../src/renderer/RenderQueue.cpp)): pure; splits the
  culled pointer list by `alphaMode`, `stable_sort`s the transparent bucket **back-to-front** (descending
  distance² from camera to `worldBounds.center()`). New mutable scratch members `opaqueItems_`/`transparentItems_`.
- **Demo** ([samples/demo/DemoApp.cpp](../samples/demo/DemoApp.cpp)): two overlapping glass panes (the plane mesh
  stood upright, 1×1 solid-colour albedo, opacity 0.4) at different depths in front of the scene; `T` toggles the
  sort (`setTransparentSortEnabled`) to expose the artifact.
- **Docs**: new [documentation/docs/22-transparency-and-alpha-blending.html](../documentation/docs/22-transparency-and-alpha-blending.html)
  (concept-first: over operator, order-dependence, depth-write-off, draw order, limitations); nav link backfilled
  across all 22 existing pages + index card. Updated ARCHITECTURE / README / ROADMAP / PROMPT.

## Verification
