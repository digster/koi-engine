# 2026-06-29 — Steps 5–8: Scene graph, textures, lighting, materials

## What we did

Completed **Step 5**. Removed the last hardcoded geometry from the renderer:
geometry now lives in reusable **`Mesh`** objects, and the flat cube list became a
**scene graph** — a tree of **`Node`**s, each with a local **`Transform`** (TRS) and
an optional shared `Mesh`. The demo scene is a small animated hierarchy (a spinning
hub cube whose satellites orbit it, one carrying its own moon) above a ground plane.
**No shader or vertex-format change** — lighting/textures are Step 6.

## Key decisions

- **Demo scene:** minimal animated hierarchy (chosen over a richer solar-system or a
  static tree) — clearest parent→child→grandchild inheritance, least demo code.
- **Layering:** `Mesh` lives in `renderer/` (it owns `SDL_GPUBuffer`s, built by the
  renderer's `createMesh` factory); `Transform`/`Node` live in `scene/`. `Node`
  references geometry via `shared_ptr<Mesh>` — an intentional `scene/ → renderer/`
  dependency. `Camera` stays SDL-free. Refines ARCHITECTURE's older "scene/ grows a
  mesh type" prediction.
- **Two-pass frame:** `Node::updateWorldTransforms` (compute `world = parentWorld ·
  local`, top-down) is kept separate from `recordScene` (draw). This is what makes
  parented motion automatic.
- **Deferred (still):** quaternions (Euler angles in `Transform`, like `Camera`); the
  `renderFrame` begin/draw/end split.

## What was built / changed

- **`math/Mat4.hpp`** — added `scaling(Vec3)` (diagonal scale, parallels `translation`).
- **`renderer/Mesh.*`** (new) — RAII owner of vertex+index buffers (+ index count) with
  a *non-owning* device pointer; non-copyable/non-movable; held via `shared_ptr`.
- **`renderer/Primitives.*`** (new) — `makeCubeMesh` (the RGB cube, moved out of the old
  `createGeometry`) and `makePlaneMesh` (flat ground quad).
- **`renderer/GpuRenderer.*`** — added `createMesh(span<Vertex>, span<Uint16>)` factory
  (reuses `uploadToGpuBuffer`); dropped the owned cube buffers + `createGeometry`;
  `isValid()` now checks only device+pipeline; `recordScene`/`renderFrame`/`captureFrame`
  take a scene root; new recursive `recordNode` binds each node's mesh + pushes
  `proj·view·world`.
- **`scene/Transform.hpp`** (new) — header-only TRS; `localMatrix() = T·R·S`.
- **`scene/Node.*`** (new) — tree node (Transform, optional shared Mesh, owned children,
  cached world matrix); `updateWorldTransforms(parentWorld)`.
- **`core/Engine.*`** — owns `unique_ptr<Node> sceneRoot_` (+ raw `hub_`/`spinner_` for
  animation); `buildScene()` assembles the hierarchy; loop spins two nodes, calls
  `updateWorldTransforms`, passes the scene to the renderer; **`shutdown()` releases the
  scene BEFORE the renderer** (mesh-before-device lifetime rule). `main.cpp` title → Step 5.
- **Tests** — `tests/test_transform.cpp` (scaling, TRS order) and `tests/test_node.cpp`
  (world-transform composition: child/grandchild stacking, parent rotation sweeps a
  child, parent scale scales offsets). All **35 tests pass** (was ~25).
- **Build** — added the 3 new sources to `koi_core` and 2 test files to `koi-tests`.
- **Docs** — new `docs/06-meshes-and-scene-graph.html` (concepts: mesh, Transform/TRS
  order, local↔world, two passes, shared meshes + group nodes, the lifetime rule);
  added the nav link to every page; `docs/index.html` card + roadmap (Step 5 ✅).
- **README / ARCHITECTURE** — status, roadmap, component table, frame data-flow,
  geometry section, a new "Meshes & the scene graph (Step 5)" section, lifecycle
  ordering, and "Planned layering" advanced to Step 6.

# Step 6 — Textures & samplers

## What we did

Completed **Step 6**: textures. We **split** the roadmap's "Textures + Phong lighting"
into two steps and did textures first (lighting is Step 7). Surfaces are now painted by
an image sampled in the fragment shader, tinted by the existing vertex color. The cubes
are checkered and the ground plane tiles the texture across the floor.

## Key decisions

- **Scope:** textures only (chosen over textures-only-vs-lighting-first-vs-both). The
  roadmap row "6 — Textures + Phong lighting" was split: Step 6 Textures, Step 7 Phong
  lighting, Models/shadows shifted to Step 8+. Updated README + index roadmaps.
- **Layering:** `Texture` lives in `renderer/` beside `Mesh` (a GPU resource, made by the
  renderer's `loadTexture`); the **sampler** is renderer-owned reusable state; the *Engine*
  owns the `Texture` and passes it to the renderer each frame (renderer stays
  asset-agnostic, like the scene root). One texture for the whole scene — per-mesh
  materials are a later step.
- **Image source:** load a real `assets/uv_grid.bmp` (committed; generated via a uv+Pillow
  script) with `SDL_LoadBMP` (no SDL_image dep), copied next to the binary by a new CMake
  `koi_assets` target. Sampler: LINEAR filter, REPEAT wrap (so the floor tiles).
- **Cube split 8 → 24 vertices:** per-face UVs can't live on shared corners (same reason
  per-face normals will need it in Step 7).

## What was built / changed

- **`Vertex`** — appended `uv[2]` (32 bytes; position/color offsets unchanged). Updated
  `static_assert`s + `tests/test_vertex.cpp` (size 32, uv@24, 24-vertex cube indices).
- **Shaders** — `triangle.vert` passes UV through; `triangle.frag` samples
  `sampler2D` at `set=2,binding=0` and does `texture(uTex,vUV) * vColor`.
- **`Shader::loadShader`** — new `numSamplers` param (sets `info.num_samplers`); fragment
  shader loaded with `numSamplers=1`.
- **`renderer/Texture.*`** (new) — RAII `SDL_GPUTexture` owner (non-owning device), like
  `Mesh`.
- **`GpuRenderer`** — `loadTexture` + `uploadToGpuTexture` (2D mirror of
  `uploadToGpuBuffer`) + `createSampler` (shared sampler); 3rd vertex attribute (UV);
  `recordScene` binds texture+sampler once via `SDL_BindGPUFragmentSamplers`;
  `renderFrame`/`captureFrame` take a `const Texture&`; dtor releases sampler; `isValid()`
  checks it.
- **`Primitives`** — cube → 24 verts with per-face UVs (kept RGB-cube tint); plane UVs
  0..6 to tile.
- **`Engine`** — owns `shared_ptr<Texture> texture_`, loads it in `init` via
  `SDL_GetBasePath()+"assets/uv_grid.bmp"`, passes it each frame, resets it before the
  renderer in `shutdown()` (texture-before-device lifetime rule). `main.cpp` title → Step 6.
- **CMake** — `Texture.cpp` in `koi_core`; `koi_assets` target copies `assets/*.bmp` →
  `build/assets/`.
- **Docs** — new `docs/07-textures-and-samplers.html` (UVs, the 24-vert split, image
  upload, samplers/filtering/wrap, `set=2` binding, sampling); nav link added to every
  page; index card + roadmap split; README + ARCHITECTURE updated (Texture row, a
  "Textures & samplers (Step 6)" section, lifecycle, `koi_assets`, planned layering → Step 7).

## Verification

- `cmake --build build` — clean under strict warnings; `koi_assets` copies the BMP;
  shaders recompiled.
- `./build/koi-tests` — **35 passed** (154 assertions, +1 for the uv offset).

# Step 7 — Phong lighting & normals

## What we did

Completed **Step 7**: lighting (the second half of the split "Textures + Phong
lighting"). Surfaces now respond to a directional light with ambient + diffuse +
specular (Blinn-Phong), modulating the Step 6 texture as albedo. Cube faces toward the
sun are bright, away are dim (ambient keeps them visible); spinning sweeps faces through
the shading.

## Key decisions

- **Light:** a single fixed **directional** light (a "sun"), white, with soft ambient
  (over point/multiple lights — those + attenuation are deferred). Blinn-Phong (halfway
  vector) over classic Phong (reflect). `shininess`/`specStrength` are shader constants.
- **Lighting in world space** ⇒ the vertex shader needs the **model matrix** separately
  from MVP. So the vertex UBO grew to `{ mvp, model }` (model = `node.worldMatrix()`,
  already computed). World normal = `mat3(model)*normal` — correct for our **uniform**
  scale; non-uniform scale would need the inverse-transpose normal matrix (deferred, noted).
- **Texture kept:** lighting multiplies the textured albedo (`texture*vColor`), it doesn't
  replace it.
- **No new geometry work:** the Step 6 24-vertex cube already gives each face its own
  vertex to carry a flat per-face normal.

## What was built / changed

- **`Vertex`** — appended `normal[3]` (44 bytes; earlier offsets unchanged). Updated
  `static_assert`s + `tests/test_vertex.cpp` (size 44, normal@32).
- **Shaders** — `triangle.vert`: `inNormal` (loc 3); UBO `{mvp, model}`; outputs
  `vWorldPos`, `vWorldNormal`. `triangle.frag`: light UBO at `set=3,binding=0`;
  ambient + diffuse(N·L) + specular(N·H) over `texture*vColor`.
- **`GpuRenderer`** — pipeline 4th vertex attribute (normal); `recordNode` pushes
  `{mvp, model}`; `recordScene` pushes the light uniform once via
  `SDL_PushGPUFragmentUniformData` and takes a `Vec3 cameraPos`; `renderFrame`/
  `captureFrame` thread `cameraPos`; fragment shader loaded with `numUniformBuffers=1,
  numSamplers=1`.
- **`Primitives`** — per-face normals on the cube's 24 verts; plane normal `+Y`.
- **`Engine`** — passes `camera_.position()` to `renderFrame`/`captureFrame`. `main.cpp`
  title → Step 7.
- **Docs** — new `docs/08-lighting-and-normals.html` (normals, world-space lighting + the
  model-matrix change, diffuse/specular/ambient, `set=3` light UBO); nav link on every page;
  index card + roadmap (Step 7 ✅); README + ARCHITECTURE updated (Vertex row, a "Lighting
  & normals (Step 7)" section, data flow, planned layering → material/Step 8).

## Verification

- `cmake --build build` — clean (after fixing a `vertexAttributes[3]`→`[4]` array size);
  shaders recompiled.
- `./build/koi-tests` — **35 passed** (155 assertions, +1 for the normal offset).
- `KOI_MAX_FRAMES=3` — runs + shuts down cleanly.
- `KOI_CAPTURE` → PNG — **directional shading confirmed on Metal**: cube tops + floor
  (normals toward the sun) bright, side faces dim (not black, ambient), texture still
  visible and lit. The two-UBO vertex stage + UBO-and-sampler fragment stage validate.

# Step 8 — Per-object materials

## What we did

Completed **Step 8**: per-object materials. Retired the single global texture +
hardcoded shininess. A `Material` (texture + shininess + specStrength) is referenced
per node and bound per draw, so objects can look different. Demo: matte checker floor,
glossy polka-dot cubes, extra-shiny hub (same dots texture, higher shininess).

## Key decisions

- **What a Material is:** `struct Material { shared_ptr<Texture> texture; float shininess;
  float specStrength; }`. Lives in `scene/` (an appearance descriptor next to `Node`),
  forward-declares `Texture`. Shared via `shared_ptr<Material>`. Completes the
  shape/placement/appearance split (Mesh / Transform / Material).
- **Binding moves per-draw:** texture bind + material params push move from once-per-frame
  (`recordScene`) into `recordNode`. Cost (rebinding even when a material is shared) is
  accepted; sort-by-material is a deferred optimization.
- **Two fragment UBOs:** light at `set=3,binding=0` (frame-constant) + material at
  `set=3,binding=1` (per-draw). `loadShader` frag now `numUniformBuffers=2, numSamplers=1`.
- **No vertex change** (materials are per-object). The global `Engine::texture_` is removed;
  textures now live in materials owned by the scene.
- **Sampler stays renderer-owned** (one shared) — a Material says *what* image, not *how*.

## What was built / changed

- **`scene/Material.hpp`** (new) — the struct (header-only; forward-declares `Texture`).
- **`scene/Node.*`** — added `shared_ptr<Material> material_` + `setMaterial`/`material()`
  + a `(mesh, material)` ctor. A node draws only with both a mesh and a material.
- **`triangle.frag`** — material UBO at `set=3,binding=1`; `material.x/.y` for
  shininess/specStrength (replacing the `kShininess`/`kSpecStrength` constants).
- **`GpuRenderer`** — `recordNode` binds the node's material texture (+ shared sampler) and
  pushes material params (frag slot 1) per draw; `recordScene` no longer binds a global
  texture; `renderFrame`/`captureFrame` dropped the `Texture&` param; frag loaded with
  `numUniformBuffers=2`.
- **`Engine`** — `buildScene` loads two textures (`uv_grid.bmp`, `dots.bmp`), builds
  floor/cube/hub materials, assigns them to nodes; removed the `texture_` member +
  threading. `main.cpp` title → Step 8.
- **`assets/dots.bmp`** (new) — a warm coral polka-dot texture (committed; generated via a
  uv+Pillow script).
- **CMake** — `koi_assets` now copies all `assets/*.bmp` via `file(GLOB)` + foreach (note:
  glob is configure-time, so new files need a CMake re-run).
- **`tests/test_node.cpp`** — added a `setMaterial`/`material()` round-trip test (null
  texture, no GPU).
- **Docs** — new `docs/09-materials.html` (shape/placement/appearance, per-draw vs per-frame
  binding, the two fragment UBOs, the demo); nav link on every page; index card + roadmap
  (Step 8 ✅, Models/shadows → 9+); README + ARCHITECTURE updated (Material row, a
  "Materials (Step 8)" section, data flow, planned layering → models).

## Verification

- `cmake --build build` — clean (after re-config picked up `dots.bmp`); both BMPs copied.
- `./build/koi-tests` — **36 passed** (159 assertions, +1 material test).
- `KOI_MAX_FRAMES=3` — loads both textures, runs, shuts down cleanly.
- `KOI_CAPTURE` → PNG (zoomed) — **distinct materials confirmed on Metal**: floor samples
  the checker, cubes sample the polka-dots (dot grid visible under the RGB tint) — proving
