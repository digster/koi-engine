# 2026-06-29 — Step 5: Meshes & a scene graph

## What we did

Completed **Step 5**. Removed the last hardcoded geometry from the renderer:
geometry now lives in reusable **`Mesh`** objects, and the flat cube list became a
**scene graph** — a tree of **`Node`**s, each with a local **`Transform`** (TRS) and
an optional shared `Mesh`. The demo scene is a small animated hierarchy (a spinning
hub cube whose satellites orbit it, one carrying its own moon) above a ground plane.
**No shader or vertex-format change** — lighting/textures are Step 6.

## Key decisions

- **Demo scene:** minimal animated hierarchy (chosen over a richer solar-system or a
  static tree) — clearest parent→child→grandchild inheritance, least demo code.
- **Layering:** `Mesh` lives in `renderer/` (it owns `SDL_GPUBuffer`s, built by the
  renderer's `createMesh` factory); `Transform`/`Node` live in `scene/`. `Node`
  references geometry via `shared_ptr<Mesh>` — an intentional `scene/ → renderer/`
  dependency. `Camera` stays SDL-free. Refines ARCHITECTURE's older "scene/ grows a
  mesh type" prediction.
- **Two-pass frame:** `Node::updateWorldTransforms` (compute `world = parentWorld ·
  local`, top-down) is kept separate from `recordScene` (draw). This is what makes
  parented motion automatic.
- **Deferred (still):** quaternions (Euler angles in `Transform`, like `Camera`); the
  `renderFrame` begin/draw/end split.

## What was built / changed

- **`math/Mat4.hpp`** — added `scaling(Vec3)` (diagonal scale, parallels `translation`).
- **`renderer/Mesh.*`** (new) — RAII owner of vertex+index buffers (+ index count) with
  a *non-owning* device pointer; non-copyable/non-movable; held via `shared_ptr`.
- **`renderer/Primitives.*`** (new) — `makeCubeMesh` (the RGB cube, moved out of the old
  `createGeometry`) and `makePlaneMesh` (flat ground quad).
- **`renderer/GpuRenderer.*`** — added `createMesh(span<Vertex>, span<Uint16>)` factory
  (reuses `uploadToGpuBuffer`); dropped the owned cube buffers + `createGeometry`;
  `isValid()` now checks only device+pipeline; `recordScene`/`renderFrame`/`captureFrame`
  take a scene root; new recursive `recordNode` binds each node's mesh + pushes
  `proj·view·world`.
- **`scene/Transform.hpp`** (new) — header-only TRS; `localMatrix() = T·R·S`.
- **`scene/Node.*`** (new) — tree node (Transform, optional shared Mesh, owned children,
  cached world matrix); `updateWorldTransforms(parentWorld)`.
- **`core/Engine.*`** — owns `unique_ptr<Node> sceneRoot_` (+ raw `hub_`/`spinner_` for
  animation); `buildScene()` assembles the hierarchy; loop spins two nodes, calls
  `updateWorldTransforms`, passes the scene to the renderer; **`shutdown()` releases the
  scene BEFORE the renderer** (mesh-before-device lifetime rule). `main.cpp` title → Step 5.
- **Tests** — `tests/test_transform.cpp` (scaling, TRS order) and `tests/test_node.cpp`
  (world-transform composition: child/grandchild stacking, parent rotation sweeps a
  child, parent scale scales offsets). All **35 tests pass** (was ~25).
- **Build** — added the 3 new sources to `koi_core` and 2 test files to `koi-tests`.
- **Docs** — new `docs/06-meshes-and-scene-graph.html` (concepts: mesh, Transform/TRS
  order, local↔world, two passes, shared meshes + group nodes, the lifetime rule);
  added the nav link to every page; `docs/index.html` card + roadmap (Step 5 ✅).
- **README / ARCHITECTURE** — status, roadmap, component table, frame data-flow,
  geometry section, a new "Meshes & the scene graph (Step 5)" section, lifecycle
  ordering, and "Planned layering" advanced to Step 6.
