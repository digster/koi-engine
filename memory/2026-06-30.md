# 2026-06-30 — Step 9: Model loading (OBJ + glTF) & shadow mapping

## What we did

Completed **Step 9**, a large two-part iteration (the user chose to bundle them):
1. **Model loading** from files via single-header libraries (per the user: use a library
   for parsing, spend the from-scratch effort on graphics). **tinyobjloader** for `.obj`,
   **cgltf** for `.glb/.gltf`. A sphere (glb) and torus (obj) now render beside the cubes.
2. **Shadow mapping**: a depth pre-pass from the light's view into a sampleable shadow map,
   sampled in the main pass so every object casts a real shadow on the floor.

## Key decisions

- **Library, not hand-rolled** (user's call): tinyobjloader + cgltf, downloaded at configure
  time like `doctest` (single headers into `build/_deps/models`). The one TU that `#define`s
  their `*_IMPLEMENTATION` (`ModelLoader.cpp`) is compiled `-w` (warnings off — third-party).
- **OBJ dedup + 32-bit indices**: OBJ indexes v/vt/vn separately → dedup each unique triple
  into one `Vertex`. Loaded models can exceed 65 535 verts → `Mesh` stores a
  `SDL_GPUIndexElementSize`; `createMesh` gained a `Uint32` overload; `recordNode` binds with
  the stored size.
- **Models generated procedurally** (no copyright): `gen_models.py` (uv+numpy+pygltflib)
  writes `assets/sphere.glb` (UV sphere) and `assets/torus.obj`, both with pos/normal/uv.
- **Shadow mapping = two passes**, single directional light. `renderShadowPass` (depth-only
  pipeline, `shadow.vert` + empty `shadow.frag`, 0 color targets) renders depth into a 2048²
  depth texture (`DEPTH_STENCIL_TARGET | SAMPLER`). Main fragment shader projects worldPos by
  `lightViewProj`, samples the map, compares with slope-scaled **bias** + **2×2 PCF**;
  occluded fragments keep ambient, lose diffuse+specular. `Mat4::orthographic` added (the
  light's frustum); `lightViewProj` computed once and used by BOTH passes.

## Bug fixed (worth remembering)

With **two** fragment samplers (material texture + shadow map), `spirv-cross` ordered the MSL
texture/sampler indices by **usage order** (shadowFactor is defined first → uShadowMap got
index 0), not by `layout(binding=)`. SDL binds by slot, so they were **swapped** on Metal — the
surface texture sampled the depth map → whole scene washed to grayscale. Fix: compile MSL with
**`--msl-decoration-binding`** so indices follow the GLSL bindings. The **capture** caught it
(invisible to the compiler). This is the same class of risk flagged since Step 6.
