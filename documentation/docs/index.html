Koi Engine Documentation

A 3D engine built from scratch, step by step, in modern C++ on top of SDL3 and its cross-platform GPU API. These docs are written for readers new to graphics programming — concepts first, then the code.

Open any page straight from disk — these docs are plain HTML with no build step. Start here:

Prerequisites, building, running, testing, and a tour of the project layout.

Window & the Render Loop →

GPUs, swapchains, command buffers and render passes explained from first principles — then mapped onto the Step 0 code.

The First Triangle →

Shaders, the graphics pipeline, clip space, and the GLSL→SPIR-V→MSL shader toolchain — then the Step 1 code that draws the gradient triangle.

Vertex & Index Buffers →

GPU vs CPU memory, staging/transfer buffers and the copy pass, vertex input layouts, and index buffers — then the Step 2 code that draws a gradient quad.

3D Cube: MVP & Depth →

Hand-rolled vector/matrix math, the Model-View-Projection chain, uniform buffers, and the depth buffer — then the Step 3 code that spins a 3D cube.

Camera & Input →

The view matrix and lookAt, a yaw/pitch fly camera, delta-time, and events vs. polled input — then the Step 4 code you fly with WASD + mouse.

Meshes & Scene Graph →

Reusable meshes, the Transform (TRS), and a tree of nodes with parent/child transforms — then the Step 5 code that spins a cube hierarchy above a ground plane.

Textures & Samplers →

Texture coordinates, uploading an image to the GPU, samplers (filtering & wrap modes), and the descriptor-set binding model — then the Step 6 code that textures the scene and tiles a floor.

Lighting & Normals →

Surface normals, a directional light, and the Phong terms (ambient, diffuse, specular) in world space — then the Step 7 code that shades the textured scene.

Materials →

A per-object Material (texture + specular params), per-draw binding, and frame-vs-object uniforms — then the Step 8 code that gives each object its own look.

Models & Shadows →

Loading OBJ/glTF models with single-header libraries, and shadow mapping (a depth pre-pass from the light) — then the Step 9 code that loads a sphere + torus and casts shadows on the floor.

Post-processing →

Off-screen HDR targets, the fullscreen-triangle trick, tone-mapping & exposure, the gamma workflow, bloom, and FXAA — then the Step 10 code that runs a fullscreen effect chain over the rendered scene.

Multiple Lights →

Directional / point / spot lights, distance attenuation, the spot cone, and accumulating many lights in a shader loop — then the Step 11 code that lights the scene with a sun plus coloured point and spot lights.

PBR Materials →

The Cook-Torrance metallic-roughness model: the microfacet BRDF (GGX, Smith, Fresnel), the metallic workflow, and energy conservation — then the Step 12 code that swaps Blinn-Phong for physically-based shading of metals and dielectrics.

Texture & Normal Maps →

Driving material parameters per-pixel from maps (metallic-roughness, AO), tangent space and the TBN matrix behind normal mapping, and mipmaps + anisotropic filtering — then the Step 13 code that adds surface relief and a per-pixel metal/dielectric checker.

Skybox & Cubemaps →

Cubemaps sampled by direction, drawing a sky as a cube around the camera, the translation-stripped view and the far-plane depth trick that seats it behind everything — then the Step 14 code that wraps the scene in a procedural day sky (and the groundwork for IBL).

Image-based Lighting →

Lighting surfaces from the environment: the diffuse irradiance map, the specular split-sum (a prefiltered environment cube plus a BRDF LUT), importance sampling and the Hammersley sequence, and baking it all by rendering into cube faces — so metals finally reflect the sky.

glTF PBR Import →

Loading a real material from a file: what glTF is, image vs. texture vs. sampler, decoding embedded PNGs with stb_image, the sRGB-vs-linear gamma trap, and an emissive term that feeds bloom — then importing the Damaged Helmet's full PBR material straight from its .glb.

Engine / App Separation →

Splitting the reusable engine from the specific app: inversion of control and the Application hooks, one FrameView bundle of "what to draw", and the shutdown ordering that frees GPU resources safely — then lifting the demo out of the engine into a sample app.

Quaternions →

Storing rotation without gimbal lock: why Euler angles jam and can't be blended, the unit quaternion as an axis + half-angle, the sandwich product that rotates a vector, the Hamilton product that composes turns, and slerp for smooth interpolation — then swapping Transform to a Quat (and why the camera keeps its yaw/pitch).
