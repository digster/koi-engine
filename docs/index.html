Koi Engine Documentation

A 3D engine built from scratch, step by step, in modern C++ on top of SDL3 and its cross-platform GPU API. These docs are written for readers new to graphics programming — concepts first, then the code.

Open any page straight from disk — these docs are plain HTML with no build step. Start here:

Prerequisites, building, running, testing, and a tour of the project layout.

GPUs, swapchains, command buffers and render passes explained from first principles — then mapped onto the Step 0 code.

The First Triangle →

Shaders, the graphics pipeline, clip space, and the GLSL→SPIR-V→MSL shader toolchain — then the Step 1 code that draws the gradient triangle.

Vertex & Index Buffers →

GPU vs CPU memory, staging/transfer buffers and the copy pass, vertex input layouts, and index buffers — then the Step 2 code that draws a gradient quad.

3D Cube: MVP & Depth →

Hand-rolled vector/matrix math, the Model-View-Projection chain, uniform buffers, and the depth buffer — then the Step 3 code that spins a 3D cube.

Camera & Input →

The view matrix and lookAt, a yaw/pitch fly camera, delta-time, and events vs. polled input — then the Step 4 code you fly with WASD + mouse.

Meshes & Scene Graph →

Reusable meshes, the Transform (TRS), and a tree of nodes with parent/child transforms — then the Step 5 code that spins a cube hierarchy above a ground plane.

Textures & Samplers →

Texture coordinates, uploading an image to the GPU, samplers (filtering & wrap modes), and the descriptor-set binding model — then the Step 6 code that textures the scene and tiles a floor.

Lighting & Normals →

Surface normals, a directional light, and the Phong terms (ambient, diffuse, specular) in world space — then the Step 7 code that shades the textured scene.
